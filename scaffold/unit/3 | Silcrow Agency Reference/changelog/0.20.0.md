# 0.20.0

**Date:** 2026-05-03

## What changed

- The plugin scaffold became **pure**: fresh agencies now start as exactly what the plugin currently IS — no fossilized supersessions, no references to previous plugin versions, no inherited drift.
- §0008 (originally "Registrar Procedural Authority", which had been superseded by §0010) was deleted from the founding set. It was a plugin-development artifact that shouldn't ship to fresh agencies.
- The founding constitutional set was renumbered from §0001–§0018 (with a gap at §0008) to a dense §0001–§0017.
- §0009 (Async Auditor) was rewritten to stand alone without the supersession framing it had inherited from §0008.
- Comprehensive prose drift cleanup: filename format normalized to `§NNNN | Title in Title Case.md`; em-dash → pipe in templates; `<role>@` → `<Role> @` (space-flanked); CANON/OPS/REFERENCE shorthand replaced with real folder names.

## Migration notes

Existing agencies scaffolded with old §-numbering will see the renumbering proposed via `:silcrow-update` as content modifications. The Registrar handles each item; User approves before execution.
