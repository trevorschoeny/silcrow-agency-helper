# 0.21.0

**Date:** 2026-05-03

## What changed

- Skills no longer search the filesystem to discover what unit they're in. The current working directory **is** the unit — its basename must start with `@` or the skill stops.
- Every fact a skill needs about the unit (agency name, user role, parent lead role) is read from a `silcrow-meta` HTML-comment anchor at the top of the unit's `README.md`, written at scaffold time:
  ```html
  <!-- silcrow-meta agency="Pebble" user-role="Trevor" lead-role="Lead" implementer-role="Implementer" -->
  ```
- `silcrow-add-unit` and `silcrow-update` validate `pwd`'s basename starts with `@` and refuse otherwise.
- `add-unit.sh` requires `--agency-name` (was previously auto-walking the tree).
- Inheritance prose across the scaffold (READMEs, AGENTS.mds, §0012, Establish Unit template, Audit Checklist, Update Workflow) describes sub-unit access as direct reference to rendered absolute paths — no walking-up language.

## Migration notes

The `silcrow-meta` anchor is required at the top of every unit's `README.md`. Existing agencies missing the anchor will see skill failures until updated; `:silcrow-update` proposes adding the anchor as a content modification.
