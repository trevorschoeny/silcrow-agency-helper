# 0.23.0

**Date:** 2026-05-03

## What changed

- **Recursive parity**: every silcrow unit, root or sub, now ships with §0001 as its local founding ADR.
  - At the agency root: §0001 is "Adopt the Silcrow Agency".
  - At sub-units: §0001 is "Adopt @ \<Parent\> as parent unit" — making §0012's federation rule explicit in the sub-unit's own canon.
- Sub-units also ship with §0002 (a unit scope seed), paralleling the agency's §0017. The Lead's first authoring exercise is to supersede §0002.
- New templates: `Adopt Parent Unit.md`, `Unit Scope.md`.
- `add-unit.sh` renders both seeds into the new sub-unit's `1 | Canon/accepted/`.
- Establish Unit template's directory-tree visualization and prose updated to reflect the seeded canon.
- Audit Checklist J extends to flag sub-units missing §0001 or §0002.

## Migration notes

Existing sub-units don't have §0001 / §0002. `:silcrow-update` proposes adding them — but at the **next available** §-numbers in each sub-unit's canon, since §0001 and §0002 in those existing canons may already be taken by past decisions. Mid-canon insertion is a structural divergence from new agencies; the audit ADR notes the difference.
