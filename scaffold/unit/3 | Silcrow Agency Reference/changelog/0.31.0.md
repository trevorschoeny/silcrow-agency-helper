# 0.31.0

**Date:** 2026-05-05

## What changed

A unified theme — **honest minimalism: silent in chat, faithful in artifacts** — codified into one new ADR, four new Message Protocol sub-rules, and dozens of targeted edits across the corpus. Two operational drift patterns are addressed:

- **Over-narration in chat.** Agents narrating routine work (archiving messages, depositing files, ADR-folder moves) and citing §-numbers in user-facing speech.
- **Over-creation in artifacts.** Agents fabricating content to populate template sections — invented "considered alternatives," generic "antipatterns," padded "out of scope" lists — even when the deliberation didn't produce them.

Both share a root cause: agents performing completeness rather than serving truth. The fix codifies the antidote.

## New canonical material

- **§0017 | Honest Minimalism** — new ADR. Establishes that every section a template defines appears in every rendered artifact (predictable structure for readers), and each section's content is either substantive or a single honest sentence (e.g., *"None considered."*, *"No anti-patterns surfaced."*, *"Review trigger: none; the failure mode this addresses is not context-dependent."*). Fabricating content to populate a slot is banned. Section headers are non-negotiable; content is bounded by what the deliberation actually produced.

- **Message Protocol §4 renamed to "Operating discipline"** with four sub-rules:
  - **§4a — Tone register split** (existing, refreshed). Agent-to-agent: verbose **and substantive**. Agent-to-user: concise. Bad/good chat examples added.
  - **§4b — Routine procedures are silent reflexes** (new). Inbox archiving, drafting, depositing, status-folder moves, README updates, citation-graph updates, loading the constitutional set on session start are silent — performed without permission, without narration, without §-citation in chat. The user only sees the work that came *after* the routine, not the routine itself.
  - **§4c — Citations live in artifacts, not chat** (new). §-numbers, MADR section anchors, and protocol-doc cross-references appear in deposited messages, ADR bodies, and commits — never in user chat. With paired bad/good examples.
  - **§4d — Faithfulness over completeness in messages** (new). The §0017 rule applies to inter-agent messages too: every section the message kind defines is present; content is substantive or single-honest-sentence. *Verbose-and-substantive*, not *verbose-and-fabricated*.

## Renumbering — important migration note

The new §0017 (Honest Minimalism) takes that slot in the seeded set. The old §0017 (Agency Scope, the seed) renumbers to **§0018** so that the scope seed remains the highest-numbered seeded ADR — its position at the bottom of the founding set is structural (it's the one expected to be superseded; everything before it is the constitutional core).

**For new agencies (`:silcrow-init` from 0.31.0+):** §0017 = Honest Minimalism, §0018 = Agency Scope (seed). Nothing to migrate.

**For existing agencies running `:silcrow-update`:** the audit will detect the shift. Two cases:

1. **Existing agency hasn't authored beyond the seeded set.** The audit proposes (a) adding a new §0017 (Honest Minimalism) ADR and (b) renaming the existing §0017 (Agency Scope) file to §0018. After approval, every reference to "§0017 (agency scope)" in inherited prose updates to §0018; the new §0017 becomes Honest Minimalism. Clean migration.

2. **Existing agency has already authored ADRs beyond §0017** (e.g., §0018, §0019 from their own decisions). The audit cannot rename §0017 → §0018 without colliding. In this case, the audit proposes adding Honest Minimalism as a *new* high-numbered ADR (e.g., §0023 if §0022 is the highest taken) rather than slotting it at §0017. The original §0017 (Agency Scope) stays put under its original number; downstream prose references are updated to point to whatever number Honest Minimalism actually got. The agency's seed-set numbering is now non-canonical compared to fresh scaffolds, but the principle is intact.

The Registrar surfaces which case applies during the audit's per-item approval pass. Read the audit report carefully.

- **§0010 (User as Principal)** — three of four antipatterns were direct negations of the decision's refinements. Replaced with a single honest sentence acknowledging the negations are stated in the refinements themselves; one genuinely new failure mode (Lead-bottleneck-on-drafts) retained.

## Migration notes

- **Existing agencies** see substantial updates from `:silcrow-update`: a new §0017 ADR (Honest Minimalism), the renumbering of Agency Scope to §0018 (or its addition at the next available number, depending on case), all four AGENTS.md edits, Message Protocol §4 restructure, MADR template updates, Decision Process worked-example additions, Foundation 04 extension, skills tightening, and the seed-ADR trims. Each item passes through per-item approval.
- **The renumbering is the most consequential mechanical change.** Run `:silcrow-update` from a clean state if possible (commit anything outstanding first); the audit's per-item approval lets you accept or defer the renumbering separately from other changes. If the renumbering creates a §-collision (case 2 above), the audit proposes the alternative numbering and surfaces the situation clearly.
- **The principle change applies immediately** once 0.31.0 is adopted. Sessions where an agent narrates routine inbox work or fabricates section content are now drift signals; cite §4b/§4d (or §0017) and push back. The chat-tone change should be especially visible — chat responses get shorter; §-citations stop appearing in user-facing speech.
