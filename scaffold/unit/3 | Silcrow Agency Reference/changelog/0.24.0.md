# 0.24.0

**Date:** 2026-05-03

## What changed

- Every ADR now carries **Agency** and **Unit** fields between Date and Authors. ADRs are self-identifying outside their filesystem context (search results, copy-paste, messages, audit reports).
- Semantic of "Unit": the unit whose `1 | Canon/` *hosts* the ADR (storage), not the unit the ADR is *about* (subject — that's in the title). So an "Establish Unit @ Pebble Core" ADR living in `@ Pebble`'s canon carries `Unit: Pebble`.
- Sub-units now ship with their own `_templates/` folder containing all ADR templates substituted with that unit's specific values. Mirrors `scaffold.sh`'s pattern for the root unit. A sub-unit Lead authoring an ADR locally finds correct Agency / Unit pre-filled.
- All 6 ADR templates and all 17 founding ADRs gained the fields.
- Registrar `AGENTS.md` drops the "or the agency's `_templates/` for sub-units that don't ship their own" fallback line.
- Audit Checklist A: new check that Agency / Unit fields are filled in (no literal `{placeholders}`).

## Migration notes

Existing ADRs predate this convention and don't have the fields. `:silcrow-update` proposes adding them as a content modification — since this is metadata addition (immutable facts being made explicit), not substantive content change, User can approve without §0004 supersession concerns.
