# 0.22.0

**Date:** 2026-05-03

## What changed

- New skill: **`:silcrow-add-agent`**. Lead-only invocation that orchestrates roster changes (§0008): authors the establishing ADR, scaffolds the agent's directory, composes its `AGENTS.md`, and proposes redistribution edits to existing agents' `AGENTS.md` as one consolidated diff before applying.
- Lead-only because redistribution is a substantive decision — §0009 prohibits the Registrar from making substantive calls. The skill verifies CWD's basename prefix matches the parent unit's `lead-role` and refuses otherwise.
- New script: `scripts/add-agent.sh` for the mechanical bits (mkdir + ADR rendering).
- New template: `Establish Agent.md` paralleling `Establish Unit.md`.
- Registrar `AGENTS.md` gains a "When a new agent is added" section.
- Audit Checklist J extends to cover agent↔ADR consistency and redistribution-narrative-vs-`AGENTS.md` mismatches.

## Migration notes

None — the new skill is additive. Existing agencies don't need any change to use it.
