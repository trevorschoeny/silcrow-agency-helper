# 0.25.0

**Date:** 2026-05-04

## What changed

- Every agent's `AGENTS.md` (Lead, Implementer, Registrar, User) now opens with a **Session start** section. Agents map the tree at startup with `find <agency_root> -type f -not -path '*/.git/*' | sort`, then load the inheritance chain: constitutional set at the agency root + ancestor canons + own unit's decisions.
- Constitutional §-number references (§0001–§0017) in Lead / Implementer / Registrar `AGENTS.md` are now qualified as `@ {agency_name}/1 | Canon/...` instead of `@ {unit_name}/...`. Sub-unit agents' citations now point at the right canon (the previous reference would fail for sub-units, since constitutional ADRs live only at the agency root).
- Walking-up-to-find-root is preserved as agent behavior — different category from skill behavior. The "no walk-up" rule was always specifically about skills doing programmatic search.
- Registrar Audit Checklist gains Step 0 ("Map the tree first") — every structural audit check operates on the listing.
- Registrar Update Workflow Step 3 (dynamic diff) now explicitly maps both plugin source and agency trees before classifying files.

## Migration notes

None — additive. Existing `AGENTS.md` files predate the Session start section; `:silcrow-update` proposes adding it as a content modification.
